Our Workflow

Although you are studying remotely for some of the time, you'll also be working on campus, in pairs, for several hours a week. To prepare you for this experience, first off you should watch and study this introduction to Pair Programming.

You'll need to use Git effectively both when working alone, to keep track of changes, and when collaborating with others. Watch this Introduction to Git and be sure to bookmark this guide to the git pairing workflow that you'll need to follow when working in pairs on sprints.

Now let's get into coding!

NodeJS

Make sure you have Node installed on your computer, and then watch Node 101.

One really important thing to learn right away is how to break up our code into different files and bring that code in elsewhere to use. Although your codebases might start off fairly small, and you might doubt the need to organise your code into different places, you'll soon see that as your projects become more complex it's essential to maintain order and readability. So let's start learning how to do this right away.

This video explains to concept of importing and exporting code with Node.

Practice importing and exporting from files until you are confident with it.

Node Package Manager

Node comes with a program called NPM (Node Package Manager) allowing us to use open souce code written by other developers. Study NPM Intro and Global Installs, followed by NPM Local Installs.

NPM Scripts

There's one final thing to know about NPM! Study our NPM Scripts video and make sure you understand how NPM scripts work and what Husky does.

Linting

We take best practices seriously at Northcoders, just as any company would. Linting (the process of checking your code for programmatic or stylistic errors) falls firmly under the category of best practices. Watch this lecture on Linting and make sure you know how to use it in your project from now onwards!
